# TopPerson 人物 Backlog

这份 backlog 是仓库的选题规划，不是最终史料定稿。
它优先选择“普通人可用”的人物方法论，保持中外平衡，并对高争议人物采取保守筛选。

这里列出的人物现在都已经在 [`.agents/skills`](../.agents/skills) 下生成了初版 skill。
下面的 `P0 / P1 / P2` 现在表示“后续打磨优先级”，而不是是否已经存在 skill 目录。

## Selection Rules

- `普通人可用优先`
  优先选择能迁移到学习、工作、带队、自律、沟通和长期决策的人物方法。
- `来源扎实优先`
  优先选择一手资料充足，或二手研究稳定可靠的人物。
- `方法重于传奇`
  优先选择能抽出稳定方法框架的人物，而不是只能讲故事的人物。
- `争议保守处理`
  对高争议、易被误用的人物，默认放入较后层级，除非实践价值明显大于风险。

## Not Prioritized Yet

- 主要依赖演义、神话、野史传播的人物
- 方法高度依赖暴力、权力或特殊时代结构的人物
- 资料过少、真实性难验证的人物
- 玄学、相术、宿命论色彩重的人物
